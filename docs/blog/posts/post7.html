Low-Specificity Best Practices

Best Practices for Writing Low-Specificity CSS

Published: November 25, 2025

The most maintainable codebases aren’t the ones with perfect specificity — they’re the ones that avoid specificity wars entirely.

Modern Techniques (2025)

  • Use :where() for base component styles
  • Prefer single-class selectors
  • Avoid selector nesting when possible
  • Use CSS custom properties for theming
  • Utility-first when appropriate

Golden Rules

  • Never use IDs in CSS
  • Never nest more than 2 levels
  • Use BEM or similar for clarity
  • Keep specificity as flat as possible

FAQ

Should I use reset or normalize?

Both — modern resets + normalize for consistency.

Write CSS that future-you will thank you for.
